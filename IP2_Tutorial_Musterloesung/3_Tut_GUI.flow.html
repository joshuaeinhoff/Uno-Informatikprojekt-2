1 GUI

Die Klasse GUI ermöglichst die Darstellung des gesamten Spiels und ist aus diesem Grund auch essenziell für die Erstellung der anderen Klassen. 

Die Konstanten dieser Klasse haben Sie bereits definiert. 

1.2 Spielbrett darstellen

Implementieren Sie:

  • eine private Prozedur, die die Lebenspunkte des Spielers darstellt
  • eine Prozedur, die die Lebenspunkte des Spielers visuell aktualisiert
  • eine Prozedur, die einen leeren Spielbrett mit den Lebenspunkten des Spielers darstellt

1.3 Menü darstellen 

In dem Menü geben Sie dem Spieler die Möglichkeit zwischen OOP1 und OOP2 zu wählen. Dies hat Einfluss auf die Fragen, die der Spieler später beantworten muss.

Implementieren Sie eine Prozedur:

  • zur Darstellung der Option "OOP1"
  • zur Darstellung der Option "OOP2" 
  • zur Darstellung der Option "OOP1", wenn ausgewählt 
  • zur Darstellung der Option "OOP2", wenn ausgewähl
  • für die Erzeugung des "GO"-Buttons
  • für die Anzeige des "GO"-Buttons
  • für das Anklicken des "GO"-Buttons
  • für die Erzeugung des grafischen Menüs

Implementieren Sie dazu noch eine Funktion zum Darstellen des grafischen Menüs, die die Auswahl des Spielniveaus zurückgibt. 

 

Hinweis

1.4 Tempel darstellen

Erstellen Sie eine Funktion zur Darstellung der einzelnen Tempel. Durch einen Klick auf das passende Symbol wird der Spieler zu einer Begrüßungsnachricht weitergeleitet. Wenn Sie möchten können sie die waitFor()- Methode benutzen, um den Schein einer Animation zu simuleren. 

Implementieren Sie eine Prozedur zur Darstellung der  Begrüßungsnachricht in jedem Tempel. Die Nachrit sollte eine kurze Einführung für die folgende Quizfrage beinhalten.

Hinweis

1.5 Quizfrage darstellen

Definieren Sie eine Funktion, die eine Quizfrage darstellt und die ausgewählte Antwort zurückgibt.

1.6 Spielfeld darstellen

Sie schreiben als erstes einige Methoden die Ihnen später bei der Zusammensetzung des Spielfelds helfen werden.

1.6.1 Darstellung der Kartenplätze 

Implementieren Sie eine Prozedur, die einen Hinweis auf den Platz der Kartenposition darstellen soll. Das kann zum Beispiel durch eine kartenförmige Umrandung realisiert werden.

1.6.2 Darstellung des Uno-Buttons

Erstellen Sie eine Prozedur, die einen Uno-Button in Abhängigkeit von der Größe des Spielfels darstellt. Der Button hat noch keine Funktion. 

1.6.3 Darstellung des Links-/Rechtsbuttons

Definieren Sie eine Prozedur zur Darstellung von zwei Buttons. Die Buttons sind für die Links- und Rechtsverschiebung der Spielerhand zuständig. Die Buttons sollten eine entsprechende Ausgabe auf die Konsole ausgeben. Die Prozedur wird durch Angabe der x- und y-Koordinate aufgerufen.

1.6.4 Darstellung des Kartenstapels

Definieren Sie eine Prozedur, die den Kartenstapel, von dem der Spieler Karten ziehen kann, darstellt. Die Vorgehensweise ähnelt der der Darstellung der Platzhalterkarten. [Kartengröße z.B. (50,100)]

1.6.5 Erstellung der Spielfelddarstellung

Implementieren Sie eine Prozedur zur Darstellung eines leeren Spielfeldes.

Gehen Sie dabei wie folgt vor: 

  • Tipp: Benutzen Sie für das Spielfeld und Hintergrund andere Farben, damit man den Unterschied erkennt. Erstellen Sie dazu noch einen kleinen sichtbaren Rand dazwischen.
  • Schaffen Sie Platz für die gespielten Karten und den Kartenstapel, indem Sie die Methode für die Platzhalterdarstellung in passenden Abständen aufrufen.
  • Für eine bessere Übersicht sollten Sie nicht alle Karten auf der Hand gleichzeitig anzeigen. (z.B. nur 10 von 20 möglichen Karten auf der Hand)
  • Stellen Sie den Kartenstapel dar. 
  • Stellen Sie alle Buttons dar

Für eine bessere Übersicht sollten Sie nicht alle Karten auf der Hand gleichzeitig anzeigen. (z.B. nur 10 von 20 möglichen Karten auf der Hand)
