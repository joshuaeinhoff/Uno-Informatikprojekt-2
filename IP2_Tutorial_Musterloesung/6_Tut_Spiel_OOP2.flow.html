1 Spiel

Bei dem letzten Teil des Tutorials beschäftigen Sie sich mit der Erstellung der Klasse, die für das ganze Spiel zuständig ist. In dieser Klasse werden die Methoden aufgerufen, die für die Entwicklung des Spiels zuständig sind.

Erstellen Sie die Klasse Spiel.

Definieren Sie die Main-Methode public static void main(String[] args).

1.1 Spielvorbereitung

Erzeugen Sie zunächst eine Instanz von GUI und Held.

Stellen Sie das Spielbrett mithilfe der Klasse GUI zum erstem Mal visuell dar.Am Anfang besteht das Spielbrett nur aus einem leeren Canvas.

Wichtig: Aus visuellen Gründen muss nach jedem Auslesen mit der Konsole das Spielbrett neu dargestellt werden.

Tipp: Geben Sie Meldungen auf die Konsole aus, um den Spielstand auch durch Text zu beschreiben.

1.2 Tempel erzeugen & Menü darstellen

Beim dem Spiel muss der Held vier Tempel überqueren, um an den Schatz zu kommen. Erzeugen Sie nun die vier verschiedenen Tempel als Instanzen der Klasse Tempel mit der entsprechenden Farbe

  • Wasser-Tempel: blau
  • Luft-Tempel: gelb
  • Erde-Tempel: grün
  • Feuer-Tempel: rot

Stellen Sie nun das Menü zur Auswahl des Spiel-Niveaus dar.

1.3 Spielablauf

Nachdem der Spieler das Spiel-Niveau durch die grafische Konsole ausgewählt hat, beginnt das Abenteuer-Spiel:

Definieren Sie zunächst eine Variable, die das Niveau bzw. die Nummer für den nächsten Tempel speichert. Achten Sie darauf, dass am Anfang noch kein Tempel ausgewählt wurde. 

Definieren Sie eine Variable zum Speichern, ob der Held gewonnen hat oder nicht. Diese Variable dient außerdem noch dazu, das Spiel fortzusetzen oder ggf. zu beenden und kann daher auch mehrmals überschrieben werden.

Jetzt folgt der eigentliche Spielablauf. Das gilt für jeden Tempel:

  • Der Spieler muss zunächst den richtigen Tempel durch die GUI auswählen. Solange der Spieler dies nicht tut, wird eine Meldung auf die Konsole ausgegeben, die den Spieler dazu auffordert den Tempel auszuwählen. Die Reihenfolge der Tempel ist dabei sehr wichtig. Der Spieler muss bei Tempel 1 (Wasser-Tempel) beginnen und sich dann "hocharbeiten" bis zum letzten Tempel (Feuer-Tempel).
  • Hat der Spieler den richtigen Tempel ausgewählt, wird nun eine Einführung in den Tempel gezeigt und das Spiel kann starten. 
  • Quiz lösen: Speichern Sie zunächst in einer Variablen ab, ob der Spieler das Quiz gelöst hat oder nicht. Sollte dies nicht der Fall sein, ist das Spiel zu Ende und die Anwendung muss abgebrochen werden.
  • Monster bekämpfen: Speichern Sie zunächst in einer Variablen ab, ob der Spieler das Monster besiegt hat. Sollte dies nicht der Fall sein, ist das Spiel zu Ende und die Anwendung muss abgebrochen werden.

Nach jedem dieser Schritte sollte das Spielbrett neu dargestellt werden!
