Einleitung

Mit Hilfe dieses Tutorials entwickeln Sie ein Abenteur-Spiel. Dabei werden nach und nach objektorientierte Eigenschaften eingesetzt. Am Ende des Tutorials können Sie das Spiel auf der Grafik-Konsole darstellen und spielen.

Temple Cards

Bei dem Spiel, das Sie programmieren werden, geht es um eine abgewandelte Version des Spiels UNO. Es handelt sich um einAbenteuer, wo der Spieler insgesamt vier Tempel (Wasser-Tempel, Luft-Tempel, Erde-Tempel, Feuer-Tempel ) überqueren muss, um das Spiel zu gewinnen und somit auch an den Schatz zu kommen. In jedem Tempel muss der Spieler Aufgaben lösen, um in den nächsten Tempel zu gelangen usw. Die zu bestehenden Aufgaben setzen sich aus einer Quizrunde und einem UNO-Spiel zusammen. Schafft der Spieler den Schatz zu finden, bekommt er als Belohnung 5Code Coins.

Die in jedem Tempel vom Spieler zu lösende Quizrunde besteht aus insgesamt 20 Fragen (5 Fragen pro Tempel) zu bestimmten OOP-Themen. Wird eine Frage richtig beantwortet, darf der Spieler weitergehen und UNO spielen. Ist dies jedoch nicht der Fall, bekommt der Spieler Schadenspunkte oder muss sogar ganz von vorne anfangen, wenn alle Fragen falsch beantwortet wurden.

1 Karte

Erstellen Sie die abstrakte Klasse Karte, die alle gemeinsamen Eigenschaften aller Karten zusammenfasst.

1.1 Daten als Instanz-Variablen definieren

Eine Karte wird durch ihre farbe und nummer beschrieben.

Definieren Sie diese Eigenschaften als Instanz-Variablen.

1.2 Superkonstruktor erstellen

Damit Sie später Objekte der Klasse Karte erzeugen können, müssen Sie an dieser Stelle einen Konstruktor erstellen. Der Konstruktor dieser Klasse ist gleichzeitig der Superkonstruktor für alle Subklassen. 

Definieren Sie den Superkonstruktor.

1.3 Prüfe, ob eine Karte spielbar ist

Implementieren Sie eine Funktion, die true zurückgibt, wenn eine Karte spielbar ist und false, wenn nicht.

Eine Karte ist spielbar, wenn:

  • ihre nummer mit der Nummer der obersten Karte auf dem Ablagestapel übereinstimmt und/oder
  • ihre farbe mit der Farbe der obersten Karte auf dem Ablagestapel übereinstimmt oder
  • die ausgewählte Karte eine Aktionskarte ist (d.h. die Farbe schwarz hat)

Die oberste Karte auf dem Ablagestapel ist die aktuelleKarte im Spiel.

1.4 Abstrakte Prozeduren und Funktionen

Deklarieren Sie:

  • die abstrakte Funktion istWelcheKarte(String kartenart), die die Kartenart zurückgibt
  • die abstrakte Prozedur erzeugeVisuelleKarte(String farbe), die die visuelle Darstellung einer Karte erzeugt und direkt danach versteckt
  • die abstrakte Prozedur setzeKarteAuf(int spalte, int zeile, int x, int y, boolean istAktuelleKarte), die die Karte auf eine gegebene Position setzt und sie sichtbar macht
  • die abstrakte Prozedur versteckeKarte(), die eine Karte versteckt
  • die abstrakte Funktion aktionDurchfuehren() zum Durchführen der Aktion einer bestimmten Karte
  • die abstrakte Funktion toString(), die eine Karte als String zurückgibt

Diese Methoden werden Sie erst in den Subklassen von Karte implementieren.

2 (Aktions-)Karten

Damit Sie UNO, wie gewohnt spielen können, müssen Sie noch verschiedene Arten von Karten definieren. Diese haben einiges gemeinsam, nämlich die Eigenschaften, die sie von der Klasse Karte erben. Sie unterscheiden sich jedoch in der Darstellung und Reaktion auf einen Klick (wenn man beispielsweise eine Aktionskarte ablegt). Da die Karten von der Klasse Karte erben, müssen Sie diese als Subklassen dieser Klasse implementieren.

Damit Sie eine bessere Vorstellung davon haben, wie die Karten am Ende aussehen sollen, stellen wir Ihnen einen Screenshot des Spiels zur Verfügung:

Bild: Bildschirmaufnahme von den Karten im Spiel

Erstellen Sie eine eigene Klasse für folgende (Aktions-)Karten: 

1. AussetzenKarte 

  • Nachdem ein Spieler diese Karte abgelegt hat, wird der  Gegenspieler „übersprungen“ und derjenige, der die Karte abgelegt hat, darf nochmal legen. 
  • Die Karte kann nur auf eine andere mit entsprechender Farbe oder eine andere AussetzenKarte gelegt werden.
  • Wenn diese Karte zu Beginn des Spiels aufgedeckt wird, muss der Spieler, der normalerweise anfangen darf, aussetzen und der Gegenspieler darf stattdessen eine Karte ablegen.

2. RetourKarte 

  • Bei dieser Karte ändert sich die Spielrichtung.
  • Die Karte kann nur auf eine entsprechende Farbe oder eine andere RetourKarte gelegt werden.
  • Wenn diese Karte zu Beginn des Spiels aufgedeckt wird, muss der Spieler, der normalerweise anfangen darf, aussetzen und der Gegenspieler darf stattdessen eine Karte ablegen.

3. PlusZweiKarte

  • Wenn diese Karte gelegt wird, muss der Gegenspieler 2 Karten ziehen und darf in dieser Runde keine Karten ablegen.
  • Diese Karte kann nur auf eine Karte mit entsprechender Farbe oder eine andere PlusZweiKarte gelegt werden.
  • Wenn sie zu Beginn des Spiels aufgedeckt wird, gelten dieselben Regeln.    

4. WunschKarte

  • Der Spieler, der diese Karte legt, entscheidet welche Farbe als nächstes gelegt werden soll. Dabei darf auch die schon liegende Farbe gewählt werden (Für die Farbwahl wird es spezielle Farb-Buttons geben, die man drücken kann, um die gewünschte Farbe auszuwählen). 
  • WunschKarten kann man immer legen, das heißt man muss nicht auf die Farbe der vorherigen Karte achten.  
  • Wenn eine WunschKarte zu Beginn des Spiels aufgedeckt wird, entscheidet der Spieler, der anfangen darf, welche Farbe als nächstes gelegt werden soll.   

5. PlusVierWunschKarte

  • Der Spieler, der diese Karte legt, entscheidet welche Farbe als nächstes gelegt werden soll. 
  • Zudem muss der Gegenspieler 4 Karten von dem Kartenstapel ziehen.  
  • Wenn sie zu Beginn des Spiels aufgedeckt wird, gelten dieselben Regeln. 

6. FarbKarte 

  • Die FarbKarte stellt eine Karte mit einer bestimmten Farbe und Nummer im Spiel dar.

7. DummyKarte

  • Die DummyKarte stellt die Karten des Gegners im Spiel dar. 

2.1 Daten als Instanz-Variablen definieren 

Damit Sie später die Karten auf der Grafikkonsole anzeigen können, müssen Sie jetzt Instanz-Variablen für die Darstellung definieren. Dafür verwenden Sie spezielle Datentypen, wie beispielsweise: Rechteck, Kreis, Linie und Text. Diese befinden sich in dem Package unotempel.grafikkonsole, was Sie für diesen Zweck auch importieren müssen.

Schauen Sie sich die Abbildung mit den Karten an und überlegen Sie erstmal, was für Elemente Sie für die Darstellung der einzelnen Karten brauchen. Definieren Sie dann in jeder einzelnen Klasse die passenden IV für die Darstellung der Karten. 

Hinweis

2.2 Konstruktor erstellen

Definieren Sie für jede Subklasse von Karte einen eigenen Konstruktor, der eine Karte mit Default Werten erzeugt. Da die Aktionskarten und die DummyKarte bei der Darstellung keine Nummer haben, setzen Sie im Konstruktor eine ein: 

  • AussetzenKarte hat die nummer 10
  • RetourKarte hat die nummer 11
  • PlusZweiKarte hat die nummer 12
  • WunschKarte hat die nummer 13
  • PlusVierWunschKarte hat die nummer 14
  • DummyKarte hat die nummer -1

Auf dieser Weise können Sie die Karten von einander unterscheiden. Sie können diese nummer also als eine Art ID sehen. 

Achtung: Bei der WunschKarte und PlusVierWunschKarte müssen Sie zwei Konstruktoren erstellen: 

  • einen Konstruktor, der eine Aktionskarte mit default-Werten (d.h. farbe "schwarz" und die entsprechende nummer) erzeugt 
  • einen Konstruktor, der eine Aktionskarte mit einer bestimmten Farbe erzeugt 

2.3 Prozeduren und Funktionen implementieren 

Sie haben bereits ein paar abstrakte Prozeduren und Funktionen in der Superklasse Karte deklariert. Diese müssen Sie nun für alle Subklassen überschreiben. 

2.3.1 Kartenart zurückgeben

Überschreiben Sie die Funktion istWelcheKarte(), die true zurückgibt, wenn die Kartenart übereinstimmt.

2.3.2 Aktion durchführen 

Überschreiben Sie für alle Karten die Funktion aktionDurchfuehren(), die für das Durchführen der Aktion einer Karte zuständig ist. Das heißt die Funktion gibt den Code für die jeweilige Aktion zurück. Jeder Code einer Aktion hat eine eigene ID. Geben Sie für folgende Karten diese ID zurück:

  • DummyKarte: 0
  • FarbKarte: 0
    • Code 0: nichts machen
  • AussetzenKarte: 1
  • RetourKarte: 1
    • Code 1: nächsten Spieler überspringen
  • PlusZweiKarte: 2
    • Code 2: zwei Karten ziehen
  • WunschKarte: 3
    • Code 3: eine neue Farbe auswählen
  • PlusVierWunschKarte: 4
    • Code 4: vier Karten ziehen und neue Farbe auswählen

Den Code für die Aktionen werden Sie im Verlauf des Tutorials programmieren. 

2.3.3 Karte als String zurückgeben

Definieren Sie für alle Subklassen die Standard-Methode toString().

2.3.4 Karte verstecken

Überschreiben Sie die Prozedur versteckeKarte(), die eine Karte versteckt. Dafür müssen Sie für alle Variablen der entsprechenden Subklasse die bereits implementierte Methode aus Grafik aufrufen. 

2.3.5 Karte auf Position setzen

Implementieren Sie in jeder Subklasse die Prozedur setzeKarteAuf(), die die Karte auf die gegebene Position setzt und sichtbar macht.

3 GUI

Die Klasse GUI ermöglicht die Darstellung des gesamten Spiels und ist aus diesem Grund auch essenziell für die Erstellung der anderen Klassen.

Für ein visuell ansprechendes Spiel ist es ratsam, sich im Vorhinein Gedanken zu einer Farbpallete zu machen. Die ausgewählten Hex-Codes können Sie, dann als Konstanten speichern.

Erstellen Sie die Klasse GUI.

3.1 Konstanten für die Darstellung des Spiels

Definieren Sie folgende Konstanten für die Darstellung des Spiels:

  • die Farbe des Spielfelds
  • die Breite des Spielfelds
  • die Höhe des Spielfelds
  • die Farben für die Darstellung der Karten

und als statische Variable:

  • ein Rechteck für die Darstellung der Lebenspunkte

In dem unten stehenden Code-Block finden Sie die Werte, die wir für die Darstelung des Spiels benutzt haben. Sie können diese so übernehmen oder nach persönlichen Vorlieben anpassen.

 

    // Farbe des Spielfelds
    public final static String FARBE_SPIELFELD = "#9CAF88";
    // Maße des Spielfelds
    public final static int BREITE = 850;
    public final static int HOEHE = 470;
    // Farben für die Darstellung der Karten
    public final static String FARBE_GRUEN = "#465726";
    public final static String FARBE_ROT = "#F57461";
    public final static String FARBE_BLAU = "#64A5BB";
    public final static String FARBE_SCHWARZ = "#232B2B";
    public final static String FARBE_GELB = "#FDD782";
    public final static String FARBE_WEISS = "#F7F9EF";
    public final static String FARBE_GRAU = "#3B444B";
    // Rechteck für die Darstellung der Lebenspunkte vom Held
    public static Rechteck lebenspunkteHeld;

 

Überschreiben Sie nun die Prozedur erzeugeVisuelleKarte() in jede Subklasse von Karte, die die visuelle Darstellung einer Karte erzeugt und sie danach direkt versteckt.

Gehen Sie wie folgt vor:

  • weisen Sie jeder Farbe die entsprechende Farb-Darstellung aus GUI zu
  • weisen Sie den bereits definierten IV passende Werte zu
  • verstecken Sie die Karte

Falls Sie die Werte, die wir für die Erzeugung der visuellen Karten benutzt haben, übernehmen möchten, können Sie das gerne tun. Klappen Sie dafür den unten stehenden Block einfach auf:)

6 Kartenspiel 

Erstellen Sie die Klasse Kartenspiel.

6.1 Daten als Instanz-Variablen definieren

Definieren Sie eine IV zum Speichern:

  • des Kartenstapels
  • des Spielfelds
  • des nächsten Spielers
    • 0 ist Held 
    • 1 ist KI (default)
  • der Spieler (Held und KI) 
  • der aktuellen Karte im Spiel 
  • der UNO-Farben

6.2 Konstruktor erstellen 

Definieren Sie einen Konstruktor, der ein Kartenspiel für zwei Spieler erzeugt und den nächsten Spieler auf den Default Wert setzt.

6.3 Ersten Spieler auswählen

Schreiben Sie eine Prozedur, die zufällig den ersten Spieler auswählt.

6.4 Aktuelle Karte setzen

Die aktuelleKarte im Spiel ist die oberste Karte auf dem Ablagestapel. Das heißt, dass die aktuelleKarte jedes Mal neu gesetzt werden muss, wenn eine neue Karte auf dem Ablagestapel abgelegt wird. 

Implementieren Sie eine Prozedur, die die aktuelleKarte im Spiel setzt und ggf. die Aktion einer Karte durchführt. Erstellen Sie eine Fallunterscheidung für die Durchführung der verschiedenen Aktionen: 

  • code 0: nichts machen 
  • code 1: den nächsten Spieler überspringen 
  • code 2: der nächtse Spieler muss zwei Karten ziehen 
  • code 3: neue Farbe auswählen 
  • code 4: neue Farbe auswählen und der nächste Spieler muss vier Karten ziehen 

6.5 Neue Farbe auswählen

Definieren Sie die Prozedur zum Auswählen einer neuen Farbe im Spiel. Achten Sie darauf, dass Sie zwischen Held (menschlicher Spieler) und KI unterscheiden müssen und geben Sie zum Schluss eine Nachricht auf die Konsole aus, damit der andere Spieler weiß, welche Farbe gewählt wurde. 

6.6 Spiel vorbereiten 

Schreiben Sie eine Prozedur zum Vorbereiten des Kartenspiels. Überlegen Sie ganz genau, welche Schritte dafür notwendig sind! 

Hints

6.7 Spielen

Implementieren Sie eine Funktion zum Spielen. Darauf sollten Sie dabei achten:

  • geben Sie immer an, welcher Spieler gerade dran ist 
  • wenn eine Karte auf dem Ablagestapel abgelegt wird, diese als aktuelleKarte setzen
  • wenn ein Spieler nur noch eine Karte auf der Hand hat, muss er auf den Button "UNO" klicken, um dem Gegenspieler das zu siganlisieren 
  • das Spiel ist zu Ende, wenn einer der Spieler keine Karten mehr auf der Hand hat 
  • der Spieler, der verloren hat, bekommt Schadenspunkte
  • zum Schluss müssen die Hände der beiden Spieler "entleert" werden, damit danach ein neues Spiel beginnen kann
