Spieler: Einleitung für OOP1

Sie haben bis jetzt alle Klassen, die Sie für das Spielen brauchen, implementiert. Aber was ist ein Spiel ohne seine Spieler? In diesem Teil des Tutorials beschäftigen Sie sich mit dem Held des Spiels und mit seinem Gegner, der KI. 

1 Spieler

Wie oben schon erwähnt, gibt es zwei Arten von Spieler: der menschliche Spieler (Held) und die KI. 

Sie benötigen, wie bei den Karten, eine Klasse, die gemeinsame Eigenschaften der Spieler zusammenfasst. 

Erstellen Sie dafür die abstrakte Superklasse Spieler und packen diese in dem entsprechenden package.

1.1 Daten als Instanz-Variablen definieren

Ein Spieler beschreiben Sie durch zwei Variablen:

  • seine Hand, mit der er die Karten "hält" 
  • seine Lebensenergie, die durch Kämpfe bzW. Spiele reduziert werden kann 

Achtung: Die IV sollen nur für Klassen, die im selben Package sind oder für die Subklassen zugreifbar sein.

Definieren Sie die zwei Instanz-Variablen. 

1.2 Superkonstruktor erstellen 

Definieren Sie den Superkonstruktor, indem Sie die oben definierten Instanz-Variablen initialisieren. Achtung: Die Hand des Spielers darf maximal 20 Karten halten, unabhängig von der Anzahl der aktuellen Karten. 

1.3 GUI ergänzen 

Da Sie nun eine Instanz von Spieler erzeugen können, müssen Sie in der Klasse GUI einige neue Sachen implementieren. Gehen Sie zum dem Tutorial für die GUI und folgen Sie den Anweisungen dort.

1.4 Abstrakte Prozeduren und Funktionen

Deklarieren Sie:

  • die abstrakte Prozedur karteZiehen(KartenStapel kartenStapel, Karte aktuelleKarte, Spielfeld spielfeld, GUI gui) zum Ziehen der Karten aus dem Kartenstapel
  • die abstrakte Funktion karteSpielen(Karte aktuelleKarte, KartenStapel karten, boolean karteGezogen, Spielfeld spielfeld,GUI gui)
    zum Spielen einer Karte
  • die abstrakte Funktion neueFarbeAuswaehlen(GUI gui) zur Auswahl einer neuen Farbe. Die Farbe wird als Nummer zurückgegeben.

Diese Methoden werden Sie erst in den Subklassen von Spieler implementieren.

1.5 Erste Hand des Spielers 

Am Anfang des Spiels bekommt jeder Spieler 7 Karten. 

Implementieren Sie die Prozedur ersteHand(KartenStapel kartenStapel, Spielfeld spielfeld, GUI gui), mit der der Spieler 7 Karten aus dem Kartenstapel ziehen kann. 

1.6 Prüfe, ob spielbare Karten vorhanden sind

Schreiben Sie die Funktion spielbareKarteVorhanden(Karte aktuelleKarte), die true zurückgibt, wenn der Spieler spielbare Karten auf der Hand hat und false, wenn das nicht der Fall sein sollte. Gehen Sie wie folgt vor:

  • iterieren Sie über die Hand des Spielers
  • wenn eine Karte am gegebenen Index vorhanden und spielbar ist, geben Sie true zurück
  • ansonsten geben Sie false zurück
Hinweis

1.7 Anzahl der Karten in der Hand zurückgeben

Schreiben Sie die Funktion anzahlKarteHand(), die die Anzahl der Karten in der Hand des Spielers zurückgibt. Gehen Sie wie folgt vor:

  • eine Variable zaehler definieren und auf 0 setzen
  • mit Hilfe einer for-each-Schleife über die Karten in der Hand des Spielers iterieren und solange eine Karte vorhanden ist, zaehler inkrementieren
  • Anzahl der Karten zurückgeben

1.8 Hand des Spielers entleeren

Es kann durchaus vorkommen, dass ein Spieler nach einem Spiel noch Karten auf der Hand hat. Aus diesem Grund müssen Sie jetzt eine Prozedur implementieren, die die Hand des Spielers nach einem Spiel "entleert".

Gehen Sie wie folgt vor:

1. Erstellen Sie eine Hilfsfunktion getKarte(), die eine in einem gegebenen Index gespeicherten Karte zurückgibt.

2. Schreiben Sie eine Prozedur karteAufNullSetzen(), die diese Karte auf null setzt.

Jetzt folgt die eigentliche Methode zum Entleeren der Hand:

Iterieren Sie über die Hand des Spielers. Solange der Spieler noch Karten auf der Hand hat, setzen Sie die Karte am gegebenen Index auf null. Die dafür benötigte Prozedur haben Sie bereits im Punkt 2 implementiert.

Implementieren Sie diese Methode.

1.9 Schaden zufügen

Wenn ein Spieler ein Spiel verliert, werden ihm Schadenspunkte hinzugefügt und somit auch die Lebensenergie reduziert.

Implementieren Sie eine Funktion, die genau das tut. 

Gehen Sie wie folgt vor:

1. Erstellen Sie eine Hilfsfunktion getLebenspunkte(), die die Lebensenergie des Spielers zurückgibt.

2. Schreiben Sie eine Prozedur, die Punkte zu der Lebensergie des Spielers addiert.

Jetzt folgt die eigentliche Methode zum Reduzieren der Lebensenergie:

  • definieren Sie eine Variable für den Schaden und setzen diese auf 0
  • iterieren Sie mit Hilfe einer for-each-Schleife über die Karten in der Hand des Spielers
  • prüfen Sie, ob eine Karte vorhanden ist
  • wenn die Karte eine "PlusVierWunsch" Karte ist, fügen Sie 5 Schadenspunkte hinzu
  • wenn die Karte eine "Wunsch" Karte ist, dann fügen sie 4 Schadenspunkte hinzu
  • wenn es eine andere Art von Karte ist, fügen sie 2 Schadenspunkte hinzu

Nun müssen Sie den Schaden für das Quiz berechnen:

  • wenn der Schaden gleich 0 ist, fügen sie 10 Schadenspunkte hinzu und ziehen diese von der Lebensenergie ab
  • falls die Lebensenergie negativ sein sollte, setzen sie diese auf 0, weil sie nicht negativ sein darf
  • zum Schluss geben Sie die Lebensenergie zurück

2 Held

Diese Klasse stellt den menschlichen Spieler dar.

Erstellen Sie die Klasse Held als Subklasse von Spieler. Vergessen Sie nicht, die Klasse in dem entsprechenden package zu packen!

2.1 Daten als Instanz-Variablen definieren  

Für die Darstellung der Karten des menschlichen Spielers müssen Sie eine Variable definieren, die den Verschiebungswert einer Karte speichert. 

2.2 Konstruktor erstellen

Definieren Sie einen Konstrukor, der einen menschlichen Spieler mit 100 Punkten Lebensenergie erzeugt und den Verschiebungswert der Karte auf 0 initialisiert. 

2.3 Eine Karte ziehen

Überschreiben Sie die Prozedur  karteZiehen(), die dafür sorgt, dass der Spieler Karten aus dem Kartenstapel zieht, solange seine Hand noch nicht voll ist.

Gehen Sie wie folgt vor:

  • iterieren Sie über die Hand des Spielers
  • überprüfen Sie, ob an einer gegebenen Stelle in der Hand eine Karte nicht vorhanden ist
  • wenn ja: muss der Spieler eine Karte aus dem Kartenstapel ziehen und an der gegebenen Stelle in der Hand, wo keine Karte vorhanden ist, hinzufügen 
  • aktualisieren Sie die Karte auf dem Spielfeld
Benutzen Sie die Methode setzeKartePosition() aus Spielfeld
  • geben Sie eine entsprechende Meldung auf die Konsole aus, wenn die Hand des Spielers voll sein sollte

2.4 Karten verschieben

Implementieren Sie eine Methode, die die Karten in der Hand des Spielers verschiebt, wenn er beispielsweise eine neue Karte zieht oder eine Karte auf dem Ablagestapel legt. 

Gehen Sie wie folgt vor:

  • definieren Sie eine Variable für das verschieben als Text
  • überprüfen Sie, ob der Spieler noch Karten auf der Hand hat und lesen Sie die Eingabe durch die Konsole
Hinweis
  • wenn verschieben gleich "links" ist und der kartenVerschiebungswert größer als 0, müssen Sie den kartenVerschiebungswert dekrementieren und anschließend die Karten verschoben darstellen
  • das gleiche gilt für "rechts", außer dass Sie da den kartenVerschiebungswert inkrementieren müssen.

2.5 Eine Karte spielen

Überschreiben Sie die Funktion karteSpielen() zum Spielen einer Karte.

Gehen Sie wie folgt vor:

1. Definieren Sie eine Variable für den Index der ausgewählten Karte.

2. Solange noch spielbare Karten in der Hand des Spielers vorhanden sind: 

  • werden die Karten verschoben (die benötigte Methode dafür, haben Sie vorhin implementiert)
  • wird eine Karte durch die Grafikkonsole ausgewählt
  • wird geprüft, ob die ausgewählte Karte gespielt werden kann 
  • wenn ja: Karte auf der Hand auf null setzen mit der bereits implementierten Methode und ausgewählte Karte zurückgeben 
  • wenn nein: Fehlermeldung mit einem entsprechenden Text auf die Konsole ausgeben 
Hinweis

3. Solange der Spieler noch keine Karte gezogen hat:

  • werden die Karten verschoben 
  • wird eine entsprechende Meldung auf die Konsole ausgegeben
  • wird geprüft, ob der Spieler auf dem Kartenstapel geklickt hat 
  • wenn ja: Karte ziehen und versuchen zu spielen 
  • wenn nein: Fehlermeldung mit einer entsprechenden Meldung, wie beispielsweise "Ziehe bitte eine Karte" auf die Konsole ausgeben 

4. Falls der Spieler nach dem Ziehen immer noch keine spielbare Karte hat, geben Sie null zurück und eine Meldung auf die Konsole aus. 

2.6 Neue Farbe Auswählen

Wie Sie bereits wissen, gibt es bei dem Kartenspiel zwei Aktionskarten (WunschKarte und PlusVierWunschKarte), mit denen man die aktuelle Farbe im Spiel wechseln kann. 

Überschreiben Sie die Funktion neueFarbeAuswaehlen(), die dem Spieler ermöglicht eine neue Farbe auszuwählen. Sie müssen hierfür nur die richtige Methode aus der Klasse GUI aufrufen. 

3 KI

Für die KI müssen Sie nun zwei Klassen erstellen:

  • KIZufall
  • KISchlau

3.1 KIZufall

Diese Klasse stellt einen einfachen KI-Gegner dar, der eher zufällig spielt.

Erstellen Sie die Klasse KIZufall als Subklasse von Spieler. Vergessen Sie nicht, die Klasse in dem entsprechendne package zu packen!

3.1.1 Konstruktor erstellen 

Definieren Sie den Konstruktor dieser Klasse, indem Sie den Superkonstruktor aufrufen.

3.1.2 Eine Karte ziehen

Diese Methode ist identisch mit der aus der Klasse Held. Der einzige Unterschied ist, dass Sie bei dem Aktualisieren der Karte auf dem Spielfeld eine DummyKarte benutzen müssen, da diese Klasse die Karten des Gegeners beschreibt und darstellt.

Implementieren Sie die Methode karteZiehen(), wie Sie das für Klasse Held gemacht haben.

3.1.3 Eine Karte spielen

Überschreiben Sie die Funktion  karteSpielen() zum Spielen einer zufällig ausgewählten Karte.

Gehen Sie wie folgt vor: 

1. Solange noch spielbare Karten in der Hand des Spielers vorhanden sind:

  • Variable für eine zufallszahl zwischen 0 und hand.length-1 definieren
  • prüfen, ob die zufällig ausgewählte Karte gespielt werden kann
  • wenn ja: Karte auf der Hand und DummyKarte auf dem Spielfeld auf null setzen mit der bereits implementierten Methode
  • ausgewählte Karte zurückgeben 

2. Solange der Spieler noch keine Karte gezogen hat:

  • eine Karte aus dem Kartenstapel ziehen
  • karteGezogen auf true setzen 
  • versuchen die gezogene Karte zu spielen 

3. Falls der Spieler nach dem Ziehen immer noch keine spielbare Karte hat, geben Sie null zurück und eine Meldung auf die Konsole aus.

3.1.4 Neue Farbe auswählen 

Überschreiben Sie die Funktion neueFarbeAuswaehlen(), die dem Spieler ermöglicht eine neue Farbe auszuwählen. Da die KI hier zufällig spielt, muss die Funktion eine zufällige Zahl zwischen 0 und 4 zurückgeben. 

3.2 KISchlau

Diese Klasse stellt einen KI-Gegner dar, der schlauer ist als die KIZufall. Das heißt, dass seine Spielzüge nicht zufällig sind.

Erstellen Sie die Klasse KISchlau als Subklasse von Spieler. Vergessen Sie nicht, die Klasse in dem entsprechenden package zu packen!

3.2.1 Konstruktor erstellen 

Definieren Sie den Konstruktor dieser Klasse, indem Sie den Superkonstruktor aufrufen.

3.2.2 Eine Karte ziehen

Implementieren Sie diese Methode genauso, wie karteZiehen() aus KIZufall.

3.2.3 Prüfen, ob eine spielbare Karte einer bestimmten Art existiert

Implementieren Sie die Funktion existiertKarte(Karte aktuelleKarte, String kartenArt), die true zurückgibt, wenn eine spielbare Karte einer bestimmten Art in der Hand des Spielers existiert und false, wenn das nicht der Fall sein sollte. 

Gehen Sie wie folgt vor:

1. Iterieren Sie über die Hand des Spielers und überprüfen:

  • ob eine Karte an einer gegebenen Index-Stelle existiert
  • ob die Karte spielbar ist 
  • welche Art von Karte das ist 
Hinweis

2. Geben Sie true zurück, wenn die oben genannten Sachen gelten, false wenn nicht.

3.2.4 Eine spielbare Karte einer bestimmten Art finden

Implementieren Sie die Funktion findKarte(Karte aktuelleKarte,String kartenArt, Spielfeld spielfeld), die eine spielbare Karte einer bestimmten Kartenart findet und zurückgibt.

Gehen Sie wie folgt vor: 

1. Iterieren Sie über die Hand des Spielers und überprüfen:

  • ob eine Karte an einer gegebenen Index-Stelle existiert
  • ob die Karte spielbar ist
  • die Kartenart 

2. Speichern Sie die ausgewählte Karte in einer Variablen.

3. Setzen Sie die Karte auf der Hand und die DummyKarte auf dem Spielfeld auf null und geben die ausgewählte Karte zurück.

4. Geben Sie null zurück, falls keine Karte gefunden wird. 

3.2.5 Karte spielen 

Überschreiben Sie die Funktion karteSpielen() zum Spielen einer Karte.

Gehen Sie wie folgt vor:

1. Prüfen Sie, ob der Spieler eine Karte einer bestimmten Art hat:

  • benutzen Sie dafür die Funktion existiertKarte()
  • wenn eine Karte einer bestimmten Art vorhanden ist, müssen Sie diese Karte finden und zurückgeben

2. Diese Schritte wiederholen Sie für alle Kartenarten. Zur Verdeutlichung können Sie sich an der Abbildung unten orientieren. 

Bild: Fallunterscheidung, Darstellung der Funktion karteSpielen()

3. Falls die KI noch keine Karte gezogen hat:

  • wird eine entsprechende Meldung auf die Konsole ausgegeben
  • muss die KI eine Karte ziehen und spielen (Funktion dafür rekursiv aufrufen) 

4. Falls die KI keine spielbare Karte auf der Hand hat:

  • wird eine entsprechende Meldung auf die Konsole ausgegeben
  • wird null zurückgegeben und der Zug ist somit beendet

3.2.6 Neue Farbe auswählen

Überschreiben Sie die Funktion neueFarbeAuswaehlen(), die dem Spieler ermöglicht eine neue Farbe auszuwählen.

Gehen Sie wie folgt vor:

1. Definieren Sie einen leeren String für die farbe und einen Array, der vier Integer (für die vier UNO-Farben) speichern kann.

2. Iterieren Sie mit einer for-each Schleife über die Karten des Spielers:

  • überprüfen Sie, ob eine Karte existiert 
  • wenn ja: die Farbe der Karte speichern 

3. Schreiben Sie eine Fallunterscheidung, bei der Sie den Index des Arrays je nach Farbe inkrementieren (Reihenfolge der Farben: blau, gelb, gruen, rot).

4. Definieren Sie zwei Variablen:

  • hoechsteAnzahl: für die höchste Anzahl von Karten und 
  • indexAufArray: für den Index der Farben / des Arrays 

und setzen diese auf -1. 

Damit die KISchlau eine Farbe auswählen kann, die Sinn macht und vorteilhaft ist, muss die KISchlau wissen, von welcher Farbe die meisten Karten vorhanden sind:  

5. Iterieren Sie über die Farben und suchen nach dem Index mit der höchsten Anzahl von Karten. Wenn die gespeicherte Zahl am gegebenen Index größer als oder gleich hoechsteAnzahl ist, dann aktaulisieren Sie hoechsteAnzahl und indexAufArray.

6. Zum Schluss geben Sie indexAufArray zurück.
