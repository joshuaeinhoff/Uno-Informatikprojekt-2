Einleitung

Mit Hilfe dieses Tutorials entwickeln Sie ein Abenteur-Spiel. Dabei werden nach und nach objektorientierte Eigenschaften eingesetzt. Am Ende des Tutorials können Sie das Spiel auf der Grafik-Konsole darstellen und spielen.

Temple Cards

Bei dem Spiel, das Sie programmieren werden, geht es um eine abgewandelte Version des Spiels UNO. Es handelt sich um einAbenteuer, wo der Spieler insgesamt vier Tempel (Wasser-Tempel, Luft-Tempel, Erde-Tempel, Feuer-Tempel ) überqueren muss, um das Spiel zu gewinnen und somit auch an den Schatz zu kommen. In jedem Tempel muss der Spieler Aufgaben lösen, um in den nächsten Tempel zu gelangen usw. Die zu bestehenden Aufgaben setzen sich aus einer Quizrunde und einem UNO-Spiel zusammen. Schafft der Spieler den Schatz zu finden, bekommt er als Belohnung 5Code Coins.

Die in jedem Tempel vom Spieler zu lösende Quizrunde besteht aus insgesamt 20 Fragen (5 Fragen pro Tempel) zu bestimmten OOP-Themen. Wird eine Frage richtig beantwortet, darf der Spieler weitergehen und UNO spielen. Ist dies jedoch nicht der Fall, bekommt der Spieler Schadenspunkte oder muss sogar ganz von vorne anfangen, wenn alle Fragen falsch beantwortet wurden.

1 Karte

Erstellen Sie die abstrakte Klasse Karte, die alle gemeinsamen Eigenschaften aller Karten zusammenfasst.

Hinweis

1.1 Daten als Instanz-Variablen definieren 

Eine Karte beschreiben Sie durch zwei Variablen:

  • farbe als Text
  • nummer als int

Definieren Sie die Instanz-Variablen.

1.2 Superkonstruktor erstellen 

Damit Sie später Objekte der Klasse Karte erzeugen können, müssen Sie an dieser Stelle einen Konstruktor definieren. Der Konstruktor der Klasse Karte ist gleichzeitig der Superkonstruktor für alle Subklassen. Das heißt, dass Sie für die Subklassen der Klasse Karte lediglich den Superkonstruktor aufrufen müssen, um Objekte erzeugen zu können.

Definieren Sie den Konstruktor der Klasse Karte, indem Sie die oben definierten Instanz-Variablen initailisieren. 

1.3 Prüfe, ob eine Karte spielbar ist

Implementieren Sie die Funktion public boolean istSpielbar(), die true zurückgibt, wenn eine Karte spielbar ist und somit abgelegt werden kann, und false, wenn das nicht der Fall sein sollte. 

Eine Karte ist spielbar, wenn:

  • ihre nummer mit der Nummer der obersten Karte auf dem Ablagestapel übereinstimmt und/oder
  • ihre farbe mit der Farbe der obersten Karte auf dem Ablagestapel übereinstimmt oder
  • die ausgewählte Karte eine Aktionskarte ist (d.h. die Farbe schwarz hat)

Die oberste Karte auf dem Ablagestapel ist die aktuelleKarte im Spiel.

Vergleichen Sie die ausgewählte Karte mit aktuellen Karte.

Hinweis

1.4 Abstrakte Prozeduren und Funktionen

Deklarieren Sie:

  • die abstrakte Funktion istWelcheKarte(String kartenart), die die Kartenart zurückgibt
  • die abstrakte Prozedur erzeugeVisuelleKarte(String farbe), die die visuelle Darstellung einer Karte erzeugt und direkt danach versteckt
  • die abstrakte Prozedur setzeKarteAuf(int spalte, int zeile, int x, int y, boolean istAktuelleKarte), die die Karte auf eine gegebene Position setzt und sie sichtbar macht
  • die abstrakte Prozedur versteckeKarte(), die eine Karte versteckt
  • die abstrakte Funktion aktionDurchfuehren() zum Durchführen der Aktion einer bestimmten Karte
  • die abstrakte Funktion toString(), die eine Karte als String zurückgibt

Diese Methoden werden Sie erst in den Subklassen von Karte implementieren.

Sie unterscheiden sich jedoch in der Darstellung und Reaktion auf einen Klick (wenn man beispielsweise eine Aktionskarte ablegt). Da die Karten von der Klasse Karte erben, müssen Sie diese als Subklassen dieser Klasse implementieren.

2.1 AussetzenKarte

Als erste Aktionskarte implementieren Sie die "Aussetzen-Karte". Diese hat folgende besondere  Eigenschaften:

  • Nachdem ein Spieler diese Karte abgelegt hat, wird der  Gegenspieler „übersprungen“ und derjenige, der die Karte abgelegt hat, darf nochmal legen. 
  • Die Karte kann nur auf eine andere mit entsprechender Farbe oder eine andere AussetzenKarte gelegt werden.
  • Wenn diese Karte zu Beginn des Spiels gezogen wird, dann muss derjenige, der anfängt, aussetzen und der Gegenspieler darf beginnen.    

Erstellen Sie die Klasse AussetzenKarte als Subklasse von Karte und geben Sie das entsprechende package an.

2.1.1 Daten als Instanz-Variablen definieren 

Damit Sie später die Karten auf der Grafikkonsole anzeigen können, müssen Sie jetzt Instanz-Variablen für die Darstellung definieren. Dafür verwenden Sie spezielle Datentypen, wie beispielsweise: Rechteck, Kreis, Linie und Text. Diese Datentypen sind Referenzdatentypen, so wie Array oder String und befinden sich in dem Package unotempel.grafikkonsole, was sie für jede Subklasse importieren müssen.

Definieren Sie folgende IV für die Darstellung einer AussetzenKarte:

  • die Kartenform als Rechteck
  • ein Kreis
  • eine Linie

Zur Veranschaulichung, warum Sie genau diese Formen für die Darstellung der Karte brauchen, können Sie sich die Abbildung oben nochmal anschauen. 

2.1.2 Konstruktor erstellen 

Damit Sie später Objekte von dieser Klasse erzeugen können, definieren Sie einen eigenen Konstruktor. Da die Subklassen von der Superklasse Karte erben, müssen Sie hier lediglich den Superkonstruktor aufrufen.

Für alle Aktionskarten setzen Sie bei dem Aufruf des Superkonstruktors einen Wert für die nummer ein:

  • AussetzenKarte hat die nummer 10.

Grund dafür ist, dass Sie auf dieser Weise einfacher mit den Aktionskarten umgehen können, da sie bei der Darstellung keine Nummer, sondern nur die Farbe schwarz haben. Sie können diese Nummer als eine Art ID betrachten.

Definieren Sie den Konstruktor der Klasse AussetzenKarte.

und als statische Variable:

5.6 Karten mischen 

Wenn ein Spiel vorbei ist, müssen die Karten neu gemischt werden. Das heißt also auch, dass Sie aktuelleKarte nicht mehr im Ablagestapel gesetzt werden darf und die abgelegteKarten wieder als vorhandeneKarten gelten.

  1. die aktuelleKarte darf nicht mehr im Kartenstapel gesetzt werden, also wird der Index davon auf -1 gesetzt 
  2. iterieren Sie über den Kartenstapel 
  3. wenn die aktuelleKarte gefunden wird, dann den Index der Karte speichern

  5. nochmal über den Kartenstapel iterieren 
  6. wenn die Karte sich im Ablagestapel befindet, aber nicht mehr die aktuelleKarte ist

  3. definieren Sie eine zufällige Kartenposition zwischen 0 und karten.length-1

  • stellen Sie die neue aktuelleKarte visuell dar 
  • erstellen Sie eine Fallunterscheidung für die Durchführung der verschiedenen Aktionen:
    • code 0: nichts machen
    • code 1: den nächsten Spieler überspringen
    • code 2: der nächtse Spieler muss zwei Karten ziehen
    • code 3: neue Farbe auswählen
    • code 4: neue Farbe auswählen und der nächste Spieler muss vier Karten ziehen

6.5 Neue Farbe auswählen

Definieren Sie die Prozedur farbeAuswaehlen(GUI gui) zum Auswählen einer neuen Farbe im Spiel. Achten Sie darauf, dass Sie zwischen Held (menschlicher Spieler) und KI unterscheiden müssen und geben Sie zum Schluss eine Nachricht auf die Konsole aus, damit der andere Spieler weiß, welche Farbe gewählt wurde.

6.6 Spiel vorbereiten

Schreiben Sie die Prozedur spielVorbereiten(Spieler held, Spieler monster, GUI gui) zum Vorbereiten des Kartenspiels. 

Gehen Sie dabei wie folgt vor: 

  • erstellen Sie ein leeres Spielfeld 
  • füllen Sie den Kartenstapel mit Karten auf 
  • bereiten Sie die einzelnen Spieler vor 
  • wählen Sie den ersten Spieler aus und geben dazu eine passende Nachricht auf die Konsole aus 
  • verteilen Sie die Karten 
  • legen Sie die erste Karte auf dem Spielfeld 
  • erstellen und füllen Sie das Spielfeld 

6.7 Spielen

Implementieren Sie die Funktion spielen(Spieler held, Spieler monster, GUI gui). Darauf sollten Sie dabei achten:

  • geben Sie immer an, welcher Spieler gerade dran ist
  • wenn eine Karte auf dem Ablagestapel abgelegt wird, diese als aktuelleKarte setzen
  • wenn ein Spieler nur noch eine Karte auf der Hand hat, muss er auf den Button "UNO" klicken, um dem Gegenspieler das zu siganlisieren
  • das Spiel ist zu Ende, wenn einer der Spieler keine Karten mehr auf der Hand hat
  • der Spieler, der verloren hat, bekommt Schadenspunkte
  • zum Schluss müssen die Hände der beiden Spieler "entleert" werden, damit danach ein neues Spiel beginnen kann
