Quiz: Einleitung für OOP1

Sie haben bis jetzt alle Klassen, die Sie für das UNO-Spiel brauchen, erstellt. Nun folgt das Quiz: 

Das Quiz besteht aus insgesamt 50 Fragen: 20 zu OOP1-Themen und 30 zu OOP2-Themen. Der Spieler hat am Anfang die Möglichkeit sich zwischen OOP1 und OOP2 zu entscheiden. Danach muss der Spieler der Reihe nach die Tempeln durchgehen (Wasser-Tempel, Luft-Tempel, Erde-Tempel, Feuer-Tempel) und Quizfragen beantworten. In jedem Tempel gibt es fünf, sieben oder acht Fragen, von denen mindestens eine richtig beantwortet werden muss, um das UNO-Spiel zu beginnen.

Wie schon erwähnt besteht das Quiz aus Fragen. Diese müssen Sie nichts selbst schreiben, sondern werden aus verschiedenen gegebenen Dateien gelesen. Das Lesen aus diesen Dateien erfolgt duch FilterReader-Subklassen, die Sie noch nicht kennen und werden daher auch für Sie zur Verfügung gestellt.

Tipp: Beachten Sie jede Klasse in ihr entsprechendes Package anzulegen!

4.3 Instanz-Methoden

4.3.1 Quizfragen auslesen

Da Sie die Quizfragen als Textdatei von uns bekommen haben, brauchen Sie jetzt eine Methode, die diese Datenstätze ausliest und ausgibt. Das machen Sie mit Hilfe eines FilterReaders. Was genau ein FilterReader ist und wie er funktioniert, lernen Sie erst in OOP2. Aus diesem Grund bekommen Sie die benötigte Methode von uns zur Verfügung gestellt. Dafür müssen Sie nur die unten stehende Notiz aufklappen. 

Die Methode quizfragenAuslesen() liest mit Hilfe eines FilterReaders je nach Tempel (bzw. Tempelfarbe) und Niveau Datensätze aus verschiedenen Dateien aus. Diese Datensätze werden am Ende als Rückgabe-Wert zurückgegeben.

Fügen Sie die folgende Methode in die Klasse Quizfragenpool ein.

/**
* Private Funktion liest, speichert und gibt Datensätze aus einer bestimmten Datei als String-Array zurück
* @param tempelFarbe - String mit der Farbe des Tempels
* @return Array von String
*/
private String[] quizfragenAuslesen(String tempelFarbe) {
  // Variablen zum Speichern der aus einer Datei gelesenen Datensätze
  String datensatz; // Zum Speichern eines Datensatzes
  String[] datensaetze = new String[50]; // d.h. Speicherplatz für max. 10 Quizfragen
  int anzahlDatensaetze = 0; // Anzahl von Datensätzen speichern und mitzählen

  // try-Block - Es wird versucht, aus einer Datei mithilfe eines FileReaders Datensätze zu lesen
  try {
    // Variable zum Speichern des Dateinamens
    String dateiName = "";
    // Je nach Farbe und Quizniveau des Tempels wird ein bestimmter Dateipfad gespeichert
    switch(tempelFarbe) {
      case "blau": // Wasser-Tempel
        if(niveau == 1)
          dateiName = "../../quizfragen/fragen_t1_oop1.txt";
                else if(niveau == 2)
                    dateiName = "../../quizfragen/fragen_t1_oop2.txt";
                break;
      case "gelb": // Luft-Tempel
        if(niveau == 1)
          dateiName = "../../quizfragen/fragen_t2_oop1.txt";
        else if(niveau == 2)
          dateiName = "../../quizfragen/fragen_t2_oop2.txt";
        break;
      case "gruen": // Erde-Tempel
                if(niveau == 1)
          dateiName = "../../quizfragen/fragen_t3_oop1.txt";
        else if(niveau == 2)
          dateiName = "../../quizfragen/fragen_t3_oop2.txt";
        break;
      case "rot": // Feuer-Tempel
        if(niveau == 1)
          dateiName = "../../quizfragen/fragen_t4_oop1.txt";
        else if(niveau == 2)
          dateiName = "../../quizfragen/fragen_t4_oop2.txt";
        break;
    }

    // Kanonische Erzeugung eines FilterReaders zum Lesen aus einer Datei
    FilterReader fr = new FilterReaderDatensatz(
                          new FilterReaderZeichen(
                          new BufferedReader(
                          new FileReader(dateiName))));

    // Variablen zum Lesen aus einer Datei
    int off = 0; // Anfang-Index für einen Array
    int len = 300; // Max. Länge von Zeichen
    int num = 0; // Anzahl von gelesenen Zeichen

    // Buffer zum Lesen und Daten aus der Datei zwischenzuspeichern
    char[] buf = new char[len + off];
    // Solange Ende des Streams nicht erreicht wird
    while (num != -1) {
      // Zeichen lesen
      num = fr.read(buf, off, len);
      // Wenn nicht Ende des Streams
      if (num != -1) {
        // String aus dem Buffer erzeugen
                datensatz = new String(buf, off, num);
                // Datensatz in den String-Array speichern und Anzahl von Datensätzen inkrementieren
                datensaetze[anzahlDatensaetze++] = datensatz;
            }
        }

  // catch-Exception - Fall IOException abgefangen wird, wird eine Meldung auf der Konsole ausgegeben
  } catch(IOException ioe) {
    ioe.getMessage();
  }
  // String-Array von Datensätzen zurückgeben
    return datensaetze;
}

4.3.2 Quizfragen bearbeiten

Mithilfe der oben angegebenen Funktion sollten Sie in der Lage sein, Datensätze aus der Datei zu lesen. Diese Datensätze müssen jetzt bearbeitet werden können. Dafür implementieren Sie die Methode quizfragenBearbeiten(String[] datensaetze), die die gelesenen Datensätze bearbeiten und als Quizfragen speichern kann.

Da die gelesenen Dateien zwischen 5 und 8 Fragen beinhalten können, müssen Sie erstmal die Anzahl der Fragen herausfinden:

1. Definieren Sie eine Variable für die Anzahl der Datensätze und setzen diese auf 0.

2. Nun iterieren Sie über die Länge der Datensätze und prüfen, ob es einen Wert an der gegebenen Index-Stelle gibt. Wenn dies der Fall sein sollte, inkrementieren Sie die Anzahl der Datensätze.

Wie Sie bereits wissen, gibt es Fragen mit drei Antwortmöglichkeiten und welche mit zwei Antwortmöglichkeiten. Aus diesem Grund müssen Sie die Anzahl der Datensätze für Quizfragen mit nur zwei Antwortmöglichkeiten anpassen.

Tipp: Sie können diese Struktur beim Öffnen eine der txt-Dateien mit den Fragen herausfinden.

Hinweis:

3. Inkrementieren Sie entsprechend die aktuelle Anzahl von Datensätzen, damit diese ein Vielfach von fünf ist.

4. Weisen Sie entsprechende Werte den Instanzvariablen zu.

5. Sie müssen nun über die Datensätze iterieren und diese in Quizfragen umwandeln. Beachten Sie, dass jede Quizfrage zwei oder drei Antwortmöglichkeiten haben kann. Wie oben schon erwähnt, können zwischen vier und fünf Datensätze eine Quizfrage erstellen. Es wird empfohlen, neue Variablen zu definieren, damit Sie diese Werte aus den Datensätze-Array zwischenspeichern können. Rufen Sie in jedem Fall den entsprechenden Konstruktor auf, um eine Quizfrage zu erstellen.

Tipp: Ein Index oder Zähler (counter) für die Datensätze kann hier hilfreich sein.

Überlegen Sie, wie man prüfen kann, ob eine Quizfrage nur zwei Antwortmöglichkeiten hat. Wie ist eine Antwort aufgebaut?

4.3.4 Quizfrage richtig beantwortet

Diese Funktion prüft, ob eine Quizfrage richtig beantwortet wurde. Wenn dies der Fall ist, wird true zurückgegeben und wenn nicht, wird false zurückgegeben.

Dafür bekommt diese Funktion eine Zufallszahl sowie eine Instanz der Klasse GUI (Grafical User Interface) übergeben. Durch das GUI sollte man eine Antwort auswählen, diese wird anschließend mit der richtigen Antwort vergliechen.

Tipp: Sie sollten die entsprechende Methode in der Klasse GUI aufrufen.
