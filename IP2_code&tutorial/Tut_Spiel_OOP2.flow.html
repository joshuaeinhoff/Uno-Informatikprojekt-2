1 Spiel

Die Klasse Spiel beinhaltet die Main-Methode und ist daher für die Ausführung der Anwendung verantwortlich. Als nächstes wird die Main-Methode ausführlich erläutert.

Erstellen Sie die Klasse Spiel.

1.1 Main-Methode

Definieren Sie die Methode main().

1. Spielvorbereitung

In der Methode müssen zuerst die Graphical User Interface und ein Spieler erzeugt werden.

Hinweis:

Zunächst muss das Spielbrett mithilfe der Klasse GUI zum ersten Mal dargestellt werden. Am Anfang besteht das Spielbrett nur aus einen leeren Canvas, wo alle Visualisierungen des Spiels dargestellt werden.

Wichtig: Aus visuellen Gründen muss nach jedem Auslesen mit der Konsole das Spielbrett neu dargestellt werden.

Tipp: Benutzen Sie Meldungen auf der Konsole durch die Klasse System. Sie helfen Ihnen, den Spielstand auch zu verbalisieren!

Beim Spiel handelt es sich um eine Schatzsuche in verschiedenen Tempeln (Wasser-, Luft-, -Erde- und Feuer-), die nun als Instanzen der Klasse Tempel mit der entsprechenden Farbe erzeugt werden müssen.

Als nächstes sollte ein Menü zur Auswahl des Spiel-Niveaus dargestellt werden.

2. Spielablauf

Nachdem der Spieler das Spiel-Niveau durch die grafische Konsole ausgewählt hat, beginnt das Abenteuer bzw. die Aktion.

Eine Variable sollte das Niveau bzw. die Nummer (von 1 bis 4) für den nächsten Tempel speichern. Definieren und initialisieren Sie diese Variable mit dem Wert 0, da der Spieler noch keinen Tempel ausgewählt hat.

Sie benötigen eine Variable zum Speichern, ob der Held eine Aufgabe im Tempel erfolgreich erledigt hat.

Tipp: Sie können diese Variable mehrmals überschreiben, da diese nur dient, um das Spiel fortsetzen oder ggf. beenden zu können.

Für jeden Tempel sollte folgendes passieren:

  • Der aktuelle Spieler muss den richtigen Tempel durch die GUI auswählen. Solange der Spieler dies nicht macht, sollte die Option zur Auswahl zur Verfügung gestellt werden. Rufen Sie die entsprechende Funktion der Klasse GUI auf.
  • Hat der Spieler den richtigen Tempel ausgewählt, wird nun eine Einführung in den Tempel gezeigt.
  • Quiz lösen: Es muss gespeichert werden, ob der Spieler das Quiz gelöst hat. Sollte dies nicht der Fall sein, ist das Spiel zu Ende und die Anwendung muss abgebrochen werden.
  • Monster Kämpfen: Es muss gespeichert werden, ob der Spieler das Monster besiegt hat. Sollte dies nicht der Fall sein, ist das Spiel zu Ende und die Anwendung muss abgebrochen werden.

Tip: Nach jedem dieser Schritte sollte das Spielbrett neu dargestellt werden.
