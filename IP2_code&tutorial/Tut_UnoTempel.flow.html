UnoTempel: Einleitung

Sie haben schon alle Klassen erstellt, sowohl für das UNO-Spiel als auch für das Quiz. Es fehlt aber noch den Kontext, den Ort, wo diese Spiele durchgeführt werden und das ist der Tempel.

Der Held bzw. die Heldin sollte jeden der vier Tempel besuchen, wo er bzw. sie bestimmte Tests bestehen muss, um letztendlich eine Belohnung zu bekommen: Einen begehrten Schatz . Dafür muss der Held zunächst in jedem Tempel sein Wissen zeigen, indem er auf eine Quizfrage richtig antwortet. Obwohl er mehrere Versuche hat, sollte er vorsichtig sein, da für jede falsch beantwortete Frage, wird er Schaden erleiden! Nachdem er eine Frage richtig beantwortet hat, muss er gegen das Monster des Tempels kämpfen, d.h. final boss, indem sie das UNO-Spiel so oft spielen, bis er das Monster besiegt hat, d.h. bis das Monster keine Lebenspunkte mehr hat. Es kann ein Spiel reichen, können sie aber mehrmals spielen. Auch hier muss der Held sich in Acht nehmen und Schaden vermeiden.

Tipp: Beachten Sie jede Klasse in ihr entsprechendes Package anzulegen!

1 Klasse Tempel

Die Klasse Tempel repräsentiert den Kontext, wo das Quiz und das UNO-Spiel durchgeführt werden. Es gibt insgesamt vier Tempel mit einem bestimmten Element (Wasser, Luft, Erde und Feuer), das durch eine Farbe (Blau, Gelb, Grün und Rot) dargestellt wird.

Erstellen Sie die Klasse Tempel.

1.1 Daten als IV definieren

Sie sollten nun die Instanz-Variablen der Klasse Tempel erstellen. Sie haben schon alle Informationen, um diese zu definieren.

Was hat ein Tempel? Sie brauchen folgendes:

  • die Farbe als Text
  • ein Quiz
  • ein Kartenspiel
  • ein Monster

1.2 Konstrukor erstellen

Der Konstruktor sollte nur eine Farbe als übergebener Parameter bekommen und die Instanz-Variablen mit Hilfe der entsprechenden Konstruktoren initialisieren.

Achtung: Kommen Sie bitte zum Punkt 1.3.1 Methode zum Erzeugen eines Monsters, wo sie die Methode monsterErzeugen() erstellen müssen, um der Konstruktor richtig zu erstellen.

Hinweis:

Erstellen Sie einen Konstruktor für die Klasse Tempel. 

1.3 Instanz-Methoden

1.3.1 Methode zum Erzeugen eines Monsters

Sie sollten nun eine Methode erstellen, um ein Monster zu erzeugen. Diese Prozedur ist nur in der Klasse Tempel sichtbar und wird im Konstruktor der Klasse aufgerufen und erzeugt je nach Tempel-Farbe eine bestimmte Art von Monster:

  • Der Wasser-Tempel hat ein Monster vom Typ KIZufall und 10 Lebenspunkte.
  • Der Luft-Tempel hat ein Monster vom Typ KIZufall und 20 Lebenspunkte.
  • Der Erde-Tempel hat ein Monster vom Typ KISchlau und 30 Lebenspunkte.
  • Der Feuer-Tempel hat ein Monster vom Typ KISchlau und 40 Lebenspunkte.

Erstellen Sie die Prozedur monsterErzeugen().

1.3.2 Methode zur Einführung in den Tempel

Um die Einführung in jeden Tempel darzustellen, benötigen Sie eine Prozedur in der Klasse Tempel, die eine Instanz von GUI als übergebener Parameter bekommt und je nach Farbe die folgende Zeichenkette aufbaut:

  • Zeile 1: Solltest du den Farbe-Tempel betreten möchten,
  • Zeile 2: musst du deine Weisheit zeigen,
  • Zeile 3: indem du ein Rätsel löst.

Achtung: Statt Farbe muss das aktuelle Element des Tempels angezeigt werden, z.B. Wasser-Tempel.

Anschließend sollte die Prozedur begruessungsnachrichtDarstellen() der Klasse GUI mit der Zeichenkette als Parameter aufgerufen werden.

Erstellen Sie die Prozedur tempelEinfuehrung().

1.3.3 Methode zum Lösen des Quizes

Nun folgt die Erstellung einer Methode, um das Quiz zu lösen. Diese Funktion gibt zurück, ob das Quiz gelöst wird.

1. Diese Funktion bekommt ein Spieler (Held), das Spielniveau (1 oder 2) sowie eine Instanz der Klasse GUI, und sollte zunächst eine Meldung auf die Konsole ausgeben, wie z.B. "Quiz lösen!". Das Spielniveau wird später in der Main-Klasse durch die GUI definiert. 

2. Danach sollte die Prozedur spielbrettDarstellen() der Klasse GUI mit dem Spieler als Parameter aufgerufen werden. 

3. Sie benötigen jetzt eine Variable, in der der Wert nach dem Aufruf der Funktion quizGeloest() der Klasse Quiz gespeichert wird. Zur Erinnerung: Diese Funktion der Klasse Quiz hat drei Parameter.

4. Sie müssen nun diese Funktion rekursiv aufrufen, wenn die erforderlichen Bedingungen zutreffen.

Hinweis: 1. Bedingung der if-Abfrage
Hinweis: 2. Bedingung der if-Abfrage

5. Es folgt eine weitere if-Abfrage für den Fall, dass der Spieler keine Lebenspunkte mehr hat. Ist dies der Fall, ist das Spiel zu Ende. Dafür muss eine Meldung auf die Konsole aufgegeben werden und zurückgegeben werden, dass das Quiz nicht gelöst wird.

6. Diese Funktion sollte am Ende zurückgeben, dass das Quiz gelöst wird.

Erstellen Sie die Funktion heldHatQuizGeloest().

1.3.4 Methode zum Vorbereiten des Kampfes

Diese Prozedur ist nur in der Klasse Tempel sichtbar und dient zur Vorbereitung des Kampfes. Sie bekommt eine Instanz der Klasse Spieler (Held) und eine Instanz der Klasse GUI als übergebene Parameter.

Es wird die Prozedur spielVorbereiten() der Klasse Kartenspiel mit den Parametern Held, Monster und GUI aufgerufen und danach wird eine Meldung auf die Konsole ausgegeben, wie bspw. "Das Monster wartet darauf... Kartenspiel ist bereit!".

Erstellen Sie die Prozedur kampfVorbereiten().

1.3.5 Methode heldHatSpielGewonnen()

Diese Funktion ist nur in der Klasse Tempel sichtbar und gibt mit dem Aufruf einer zweiten Funktion zurück, ob der Held das Spiel gewonnen hat. Dafür werden eine Instanz der Klasse Spieler (Held) und einer Instanz der Klasse GUI als Parameter übergeben. Es wird eine Meldung auf die Konsole ausgegeben, wie z.B. "Spielen!". 

Letztendlich wird die Methode spielen() der Klasse Kartenspiel mit den entsprechenden Parametern aufgerufen.

Erstellen Sie die Funktion heldHatSpielGewonnen().

1.3.6 Methode heldHatMonsterBesiegt()

Diese Methode gibt zurück, ob der Held das Monster besiegt hat. Außerdem bekommt diese Funktion als Parameter:

  • Eine Instanz von Spieler (Held)
  • Ob der Held das Spiel gewonnen hat
  • Eine Instanz der Klasse GUI

Es muss nun geprüft werden, dass beide Spieler weiter spielen können. Das ist nur der Fall, wenn die beiden noch Lebenspunkte haben.

1. Zunächst wird eine Meldung auf die Konsole ausgegeben, wie z.B."Held muss das Monster besiegen, indem er UNO spielt.".

2. Danach sollte die Prozedur kampVorbereiten() mit den entsprechenden Parametern aufgerufen werden.

3. Es muss nun der Parameter überschrieben werden, wo gespeichert wird, ob der Held das Spiel gewonnen hat. Zunächst wird den Aufruf auf die Funktion heldHatSpielGewonnen() hier gespeichert.

4. Die Funktion sollte rekursiv aufgerufen werden, diesmal mit dem neuen oben gespeicherten Wert.

Das Spiel muss beendet werden, wenn die erforderliche Bedingung zutrifft. Ist dies der Fall, sollte klargemacht werden, dass der Held das Monster nicht besiegt hat.

Abschließend wird zurückgegeben, dass der Held das Monster besiegt hat.

Erstellen Sie die Funktion heldHatMonsterBesiegt().
