UNO: Einleitung für OOP1

Mit Hilfe dieses Tutorials entwickeln Sie Schritt für Schritt ein Spiel. Dabei werden nach und nach objektorientierte Eigenschaften eingesetzt. Außerdem können Sie das Spiel am Ende des Tutorials auf der Grafik-Konsole darstellen und spielen.

Spielkonzept

Bei dem Spiel, das Sie entwickeln werden, geht es um eine abgewandelte Version des Spiels UNO. Es handelt sich um eine Schatzsuche, wo der Spieler insgesamt vier Tempeln (Wasser-Tempel, Luft-Tempel, Erde-Tempel, Feuer-Tempel ) überqueren muss, um zu gewinnen und somit auch an den Schatz zu kommen. In jedem Tempel muss der Spieler Aufgaben lösen, um in den nächsten Tempel zu gelangen usw. Die zu bestehenden Aufgaben setzen sich aus einer Quizrunde und einem UNO-Spiel zusammen. Schafft der Spieler den Schatz zu finden, bekommt er als Belohnung 5Code Coins.

Die in jedem Tempel vom Spieler zu lösende Quizrunde besteht aus insgesamt 20 Fragen (5 Fragen pro Tempel) zu bestimmten OOP-Themen. Der Spieler hat x Sekunden Zeit die Frage zu beantworten. Wird die Frage richtig beantwortet, darf der Spieler weitergehen und UNO spielen. Ist dies jedoch nicht der Fall, bekommt der Spieler Schadenspunkte oder muss sogar ganz von vorne anfangen, wenn alle Fragen falsch beantwortet wurden.

Sie fangen erstmal mit dem UNO-Tempel Kartenspiel an. Zur Veranschaulichung der Packages und der ganzen Klassen, stellen wir Ihnen Diagramme zur Verfügung.

1 Karte

Erstellen Sie als erstes die abstrakte Klasse Karte, die alle gemeinsamen Eigenschaften aller Karten zusammenfasst.

Hinweis

1.1 Daten als Instanz-Variablen definieren 

Eine Karte beschreiben Sie durch zwei Variablen:

  • farbe als String 
  • nummer als int

Definieren Sie die Instanz-Variablen.

1.2 Superkonstruktor erstellen 

Damit Sie später Objekte der Klasse Karte erzeugen können, müssen Sie an dieser Stelle einen Konstruktor definieren. Der Konstruktor der Klasse Karte ist gleichzeitig der Superkonstruktor für alle Subklassen. Das heißt, dass Sie für die Subklassen der Klasse Karte lediglich den Superkonstruktor aufrufen müssen, um Objekte erzeugen zu können.

Definieren Sie den Konstruktor der Klasse Karte, indem Sie die oben definierten Instanz-Variablen initailisieren. 

1.3 Prüfe, ob eine Karte spielbar ist

Implementieren Sie die Funktion public boolean istSpielbar(), die true zurückgibt, wenn eine Karte spielbar ist und somit abgelegt werden kann, und false, wenn das nicht der Fall sein sollte. 

Eine Karte ist spielbar, wenn:

  • ihre nummer mit der Nummer der obersten Karte auf dem Ablagestapel übereinstimmt und/oder
  • ihre farbe mit der Farbe der obersten Karte auf dem Ablagestapel übereinstimmt oder
  • die ausgewählte Karte eine Aktionskarte ist (d.h. die Farbe schwarz hat)

Vergleichen Sie die ausgewählte Karte mit der obersten Karte auf dem Ablagestapel.

Hinweis

1.4 Abstrakte Prozeduren und Funktionen

Deklarieren Sie:

  • die abstrakte Funktion istWelcheKarte(String kartenart), die true zurückgibt, wenn die Kartenart übereinstimmt, sonst false 
  • die abstrakte Prozedur erzeugeVisuelleKarte(String farbe), die die visuelle Darstellung einer Karte erzeugt und direkt danach versteckt
  • die abstrakte Prozedur setzeKarteAuf(int spalte, int zeile, int x, int y, boolean istAktuelleKarte), die die Karte auf eine gegebene Position setzt und sie sichtbar macht
  • die abstrakte Prozedur versteckeKarte(), die eine Karte versteckt
  • die abstrakte Funktion aktionDurchfuehren() zum Durchführen der Aktion einer bestimmten Karte
  • die abstrakte Funktion toString(), die die Karte als String zurückgibt

(!) Diese Methoden werden Sie erst in den Subklassen von Karte implementieren.

3.1 AussetzenKarte

Als erste Aktionskarte implementieren Sie die "Aussetzen-Karte". Diese hat folgende besondere  Eigenschaften:

  • Nachdem ein Spieler diese Karte abgelegt hat, wird der  Gegenspieler „übersprungen“ und derjenige, der die Karte abgelegt hat, darf nochmal legen. 
  • Die Karte kann nur auf eine andere mit entsprechender Farbe oder eine andere AussetzenKarte gelegt werden.
  • Wenn diese Karte zu Beginn des Spiels gezogen wird, dann muss derjenige, der anfängt, aussetzen und der Gegenspieler darf beginnen.    

Erstellen Sie die Klasse AussetzenKarte als Subklasse von Karte und geben Sie das entsprechende package an.

Hinweis

Definieren Sie folgene IV für die Darstellung einer AussetzenKarte:

Damit Sie später Objekte von dieser Klasse erzeugen können, definieren Sie einen eigenen Konstruktor. Da die Subklassen von der Superklasse Karte erben, müssen Sie hier lediglich den Superkonstruktor aufrufen.

Grund dafür ist, dass wir auf dieser Weise einfacher mit den Aktionskarten umgehen können, da sie bei der Darstellung keine Nummer, sondern nur die Farbe schwarz haben. Sie können diese Nummer als eine Art ID betrachten.

3.1.5 Karte auf Position setzen

Überschreiben Sie die Prozedur setzeKarteAuf(), die die Karte auf die gegebene Position setzt und sichtbar macht. Fügen Sie dafür den unten stehen Code an der richtigen Position ein: 

  /**
     * Prozedur setzt die Karte auf die gegebene Position und macht diese sichtbar
     * @param spalte - Koordinate auf der X-Achse im Spielfeld (2x2-KarteArray)
     * @param zeile - Koordinate auf der Y-Achse im Spielfeld (2x2-KarteArray)
     * @param x - Koordinate auf der X-Achse für die Darstellung
     * @param y - Koordinate auf der Y-Achse für die Darstellung
     * @param istAktuelleKarte - true, wenn die Karte die aktuelle Karte im Spiel entspricht - false, sonst
     */
    @Override
    public void setzeKarteAuf(int spalte, int zeile, int x, int y, boolean istAktuelleKarte) {
        /* // Warte 2 Sekunden darauf
        this.karteform.waitFor(200);
        this.kreis.waitFor(200);
        this.linie.waitFor(200); */
        // Variablen verschieben
        this.karteform.moveTo(x,y);
        this.kreis.moveTo(x+10,y+35);
        this.linie.moveTo(x+5,y+35);
        // Variablen anzeigen
        this.karteform.show();
        this.kreis.show();
        this.linie.show();
        // Prüfen, ob die Karte nicht die aktuelle Karte ist
        if(!istAktuelleKarte) {
          // Reaktion einsetzen
            this.karteform.setCallback(spalte+"");
            this.kreis.setCallback(spalte+"");
            this.linie.setCallback(spalte+"");
        } else {
          // Reaktion aussetzen
            this.karteform.setCallback("");
            this.kreis.setCallback("");
            this.linie.setCallback("");
          }    
    }

6.6 Karten mischen 

Wenn ein Spiel vorbei ist, müssen die Karten neu gemischt werden. Das heißt also auch, dass Sie aktuelleKarte nicht mehr im Ablagestapel gesetzt werden darf und die abgelegteKarten wieder als vorhandeneKarten gelten.

  2. iterieren Sie über den Kartenstapel 
  3. wenn die aktuelleKarte gefunden wird, dann den Index der Karte speichern
  4. Schleife unterbrechen 
  5. nochmal über den Kartenstapel iterieren 
  6. wenn die Karte sich im Ablagestapel befindet, aber nicht mehr die aktuelleKarte ist
