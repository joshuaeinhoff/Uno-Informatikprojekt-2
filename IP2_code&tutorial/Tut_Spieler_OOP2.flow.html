Spieler: Einleitung für OOP2

Sie haben bis jetzt alle Klassen, die Sie für das Spielen brauchen, implementiert. Aber was ist ein Spiel ohne seine Spieler? In diesem Teil des Tutorials beschäftigen Sie sich mit dem Held des Spiels und mit seinem Gegner, der KI.

1 Spieler

Wie oben schon erwähnt, gibt es zwei Arten von Spieler: der menschliche Spieler (Held) und die KI.

Sie benötigen, wie bei den Karten, eine Klasse, die gemeinsame Eigenschaften der Spieler zusammenfasst.

Implementieren Sie die abstrakte Superklasse Spieler.

1.1 Daten als IV implementieren

Ein Spieler beschreiben Sie durch zwei Variablen:

  • seine Hand, mit dem er die Karten "hält"
  • seine Lebensenergie, die durch Kämpfe bzW. Spiele reduziert werden kann

Die IV sollen nur für Klassen, die im selben Package sind oder für die Subklassen zugreifbar sein.

Definieren Sie die Instanz-Variablen.

1.2 Superkonstrukor definieren

Initialisieren Sie die oben definierten IV im Konstruktor. Die Hand hat immer die Länge 20, unabhängig von der Anzahl der aktuellen Karten.

Definieren Sie den Superkonstruktor für die Klasse Spieler.

1.3 Erste Hand des Spielers

Am Anfang des Spiels müssen beide Spieler 7 Karten aus dem Kartenstapel ziehen.

Implementieren Sie dafür die Methode public void ersteHand(KartenStapel kartenStapel, Spielfeld spielfeld, GUI gui), die insgsamt 7 Karten aus dem Kartenstapel zieht. Die benögtigte Methode dafür, haben Sie schon definiert (Hinweis: die aktuelleKarte muss am Anfang auf null gesetzt werden).

Geben Sie eine Meldung auf die Konsole aus, nachdem der Spieler die Karten gezogen hat und seine erste Hand somit "erzeugt wurde".

1.4 Hand des Spielers entleeren

Es kann durchaus vorkommen, dass ein Spieler nach einem Spiel noch Karten auf der Hand hat. Aus diesem Grund müssen Sie jetzt eine Methode implementieren, die die Hand des Spielers nach einem Spiel "entleert".

Gehen Sie wie folgt vor:

1. Schreiben Sie eine Hilfsfunktion getKarte(), die eine in einem gegebenen Index gespeicherten Karte zurückgibt.

2. Schreiben Sie eine Prozedur karteAufNullSetzen(), die diese Karte auf null setzt.

Jetzt folgt die eigentliche Methode zum Entleeren der Hand:

Iterieren Sie über die Hand des Spielers. Solange der Spieler noch Karten auf der Hand hat, setzten Sie die Karte am gegebenen Index auf null. Dafür rufen Sie einfach die im Punkt 2. definierte Prozedur auf.

Implementieren Sie diese Methode.

1.5 Anzahl der Karten in der Hand zurückgeben

Schreiben Sie die Funktion anzahlKarteHand(), die die Anzahl der Karten in der Hand des Spielers zurückgeben soll. Gehen Sie wie folgt vor:

  • eine Variable zaehler definieren und auf 0 setzen
  • mit Hilfe einer for-each-Schleife über die Karten in der Hand iterieren und solange eine Karte vorhanden ist, zaehler inkrementieren
  • Anzahl der Karten zurückgeben

1.6 Sind spielbare Karten vorhanden?

Schreiben Sie die Funktion public boolean spielbareKarteVorhanden(Karte aktuelleKarte), die true zurückgibt, wenn der Spieler spielbare Karten auf der Hand hat und false, wenn das nicht der Fall sein sollte. Gehen Sie wie folgt vor:

  • iterieren Sie über die Hand des Spielers
  • wenn eine Karte am gegebenen Index vorhanden und spielbar ist (Hinweis: prüfen Sie dies mit der Methode istSpielbar() aus Karte), geben die true zurück
  • und wenn das nicht der Fall sein sollte false

1.7 Schaden zufügen

Schreiben Sie die Funktion schadenZufuegen(), die die Lebensenergie des Spielers reduzieren soll.

Gehen Sie wie folgt vor:

1. Schreiben Sie eine Hilfsfunktion getLebenspunkte(), die die Lebensenergie des Spielers zurückgibt.

2. Definieren Sie die Prozedur lebenspunkteGewinnen(int punkte), die Punkte zu der Lebensergie des Spielers addiert.

Jetzt folgt die eigentliche Methode zum Reduzieren der Lebensenergie:

  • definieren Sie eine Variable für den Schaden und setzen diese auf 0
  • iterieren Sie mit Hilfe einer for-each-Schleife über die Karten in der Hand des Spielers
  • prüfen Sie, ob eine Karte vorhanden ist
  • wenn die Karte eine "PlusVierWunsch" Karte ist, fügen Sie 5 Schadenspunkte hinzu
  • wenn die Karte eine "Wunsch" Karte ist, dann fügen sie 4 Schadenspunkte hinzu
  • wenn es eine andere Art von Karte ist, fügen sie 2 Schadenspunkte hinzu

Nun müssen Sie den Schaden für das Quiz berechnen:

  • wenn der Schaden gleich 0 ist, fügen sie 10 Schadenspunkte hinzu und ziehen diese von der Lebensenergie ab
  • falls die Lebensenergie negativ sein sollte, setzen sie diese auf 0
  • zum Schluss geben Sie die Lebensenergie zurück

Implementieren Sie diese Methode.

2 Held

Diese Klasse stellt den menschlichen Spieler dar.

Implementieren Sie die Klasse Held als Subklasse von Spieler.

2.1 Daten als IV implementieren

Definieren Sie eine Variable, die den Verschiebungswert einer Karte speichert.

2.2 Konstruktor definieren

Definieren Sie den Konstrukor dieser Klasse, indem Sie den Superkonstruktor aufrufen, einen menschlichen Spieler mit 100 Punkten Lebensenergie erzeugen und die IV für den Verschiebungswert der Karte auf 0 setzten.

2.3 Eine Karte ziehen

Implementieren Sie die Methode public void karteZiehen(KartenStapel kartenStapel, Karte aktuelleKarte, Spielfeld spielfeld, GUI gui), die dafür sorgt, dass der Spieler Karten aus dem Kartenstapel zieht, solange seine Hand noch nicht voll ist. Vergessen Sie nicht am Ende den Kartenstapel zu aktualisieren. Falls die Hand des Spielers voll sein sollte, geben Sie eine entsprechende Meldung auf die Konsole aus.

2.4 Karten verschieben

Implementieren Sie eine Methode, die die Karten in der Hand des Spielers verschiebt, wenn er beispielsweise eine neue Karte zieht oder eine Karte auf dem Kartenstapel ablegt. Gehen Sie wie folgt vor:

1. Definieren Sie eine Variable für das Verschieben einer Karte als Text.

2. Prüfen Sie, ob der Held noch Karten auf der Hand hat und lesen Sie die Eingabe durch die Konsole.

3. Wenn die IV verschieben gleich "links" ist und der kartenVerschiebungswert größer als 0, müssen Sie den kartenVerschiebungswert dekrementieren und anschließend die Karten verschoben darstellen. Das gleiche gilt für "rechts", außer dass Sie da den Wert inkrementieren müssen.

2.5 Eine Karte spielen

Implementieren Sie die Methode public Karte karteSpielen(Karte aktuelleKarte, KartenStapel kartenStapel, boolean karteGezogen,Spielfeld spielfeld,GUI gui) wie folgt:

1. Definieren Sie eine Variable indexKarte für den Index der ausgewählten Karte.

2. Solange die aktuelle Karte spielbar ist, müssen die Karten in der Hand verschoben werden. Die benögtigte Methode dafür, haben Sie schon definiert.

3. Nur falls indexKarte ungleich -1 ist, prüfen Sie ob die ausgewählte Karte gespielt werden kann. Wenn dies der Fall sein sollte, speichern Sie die ausgewählte, spielbare Karte als Variable und setzen dann die Karte auf der Hand auf null, da sie jetzt in der Hand nicht mehr vorhanden ist. Zum Schluss geben Sie die ausgewählte Karte zurück.

Geben Sie eine Meldung auf die Konsole aus, wenn die ausgewählte Karte nicht spielbar sein sollte.

4. Wenn der Spieler keine spielbare Karte hat, muss er eine aus dem Kartenstapel ziehen. Es wird erstmal geprüft, ob der Spieler schon eine Karte gezogen hat. Das machen Sie indem Sie prüfen, ob der Spieler auf den Kartenstapel geklickt hat. Wenn die gezogene Karte spielbar ist, darf der Spieler sie legen. Das machen Sie mit einem rekursiven Aufruf der Methode karteSpielen().

Geben Sie eine Meldung auf die Konsole aus, wenn der Spieler keine spielbare Karte haben sollte.

2.6 Neue Farbe Auswählen

Wie Sie bereits wissen, gibt es in dem UNO-Spiel ein paar Aktionskarten, wie z.B. die WunschKarte. Der Spieler, der diese Karte legt, entscheidet welche Farbe als nächstes gelegt werden soll.

Schreiben Sie eine Funktion, die dem Spieler ermöglicht eine neue Farbe auszuwählen. Sie müssen hierfür nur die richtige Methode aus der Klasse GUI aufrufen.
